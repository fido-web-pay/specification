FIDO® Web Pay - Introduction
It would not be an exaggeration claiming that EMV® represents the current "Gold Standard" for secure and convenient payments in the physical world.
This document describes FIDO Web Pay (FWP), which is a Web-adapted version of EMV, building on FIDO2 technology combined with W3C's PaymentRequest API.
However, unlike EMV, FWP is intended to be usable with any account-based payment system, including the eurozone's SEPA Inst.
Also in contrast to EMV, FWP encrypts user-authorizations, which has the following side effects:
  • Eliminating dependencies on external tokenization services.
  • Limiting the amount of personally identifiable information (PII), shared with third-parties to the name of the payer's bank.
  • Eliminating PCI requirements with respect to the payer.
Although not a part of this specification, the underpinning authorization scheme also opens the door to non-direct payments, including secure (tokenized) versions of Card-on-File.
Prerequisites
The scheme outlined here depends on that the User already has received a payment credential from an Issuer in an enrollment process. A payment credential is supposed to at least contain the following elements:
  • A User specific (unique) FIDO key for signing payment authorization data.
  • An Issuer specific (shared) public key for encrypting payment authorization data.
  • An accountId associated with the FIDO signature key.
  • A paymentMethod URL identifying the payment ecosystem or network.
  • An issuerId in the form of a BIN number or a URL associated with the Issuer.
  • An icon identifying the payment credential for the User.
Note that FWP effectively forms a browser-resident, universal "wallet", holding any number of payment credentials.
The exact enrollment process is yet to be defined; the assumption is that it could be accomplished by extending the existing W3C WebAuthn enrollment system.
